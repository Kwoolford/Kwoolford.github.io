# kwoolford.github.io

Personal site ‚Äî a research notebook of projects, notes, and digests. Published at <https://kwoolford.github.io>.

Built with Jekyll using a custom local theme (no external theme gem). GitHub Pages builds and publishes automatically on push.

## Layout

```
_config.yml               # site settings
_data/now.yml             # editable "now" block shown on the homepage
_layouts/
  default.html            # base HTML
  home.html               # hero / now / featured / archive / notes / meta panel
  post.html               # project + note page
  page.html               # generic static page
_includes/                # head, nav, footer, project-card, brand-mark
assets/
  css/main.scss           # single stylesheet (design tokens + Rouge palette)
  js/search.js            # client-side archive filter
_posts/YYYY-MM-DD-*.md    # entries
archive.md                # full filterable archive at /archive/
```

## Entry frontmatter

Every entry is a post under `_posts/`. The theme reads these fields:

| Field | Required | Effect |
|------|----------|--------|
| `title` | yes | heading |
| `date` | yes (via filename) | timestamp |
| `categories` | yes | `[project]` or `[note]` ‚Äî controls where it appears on the homepage |
| `tags` | no | shown on cards, feeds RSS |
| `stack` | no | pill row (falls back to `tags`) |
| `status` | no | `active` / `paused` / `done` / `experimenting` ‚Üí colored status chip |
| `summary` | no | one-sentence lede on card + top of page |
| `featured` | no | `true` puts it in the homepage "featured" section |
| `project_path` | no | absolute project path shown in the page metadata |
| `autogenerated` | no | adds an "autogenerated digest" pill to the page footer |

Example:

```markdown
---
title: My Project
date: 2026-04-18
categories: [project]
tags: [rust, cli]
stack: [rust, tokio, clap]
status: active
summary: >
  One-sentence description of what the project does.
featured: true
---

Opening paragraph here.

## Purpose
...

## Highlights
...

## Links
- [github.com/Kwoolford/my-project](https://github.com/Kwoolford/my-project)
```

## Adding a post via Claude Code

Run the `project-digest` skill from your projects folder:

- `/project-digest` ‚Äî pick a project interactively
- `/project-digest ./my-project` ‚Äî digest one project
- `/project-digest all` ‚Äî iterate every project folder

The skill reads each project's README, emits the frontmatter fields above, writes a digest into `_posts/`, and (with confirmation) commits and pushes.

```bash
bundle exec jekyll serve
```

Visit <http://localhost:4000>.
