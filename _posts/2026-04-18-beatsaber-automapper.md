An open-source AI system that takes an audio file and produces a playable Beat Saber level — notes, arcs, chains, bombs, obstacles, and a synchronized light show — packaged as a v3-format `.zip`. The pipeline is a three-stage transformer: a shared audio encoder feeds an onset detector, a note-sequence decoder, and a lighting decoder, each conditioned on difficulty, genre, and per-frame song-structure features. The current line of work ("V5") has moved away from a single averaged model to per-mapper style cohorts, with an auto-researcher harness that runs a YAML queue of training specs overnight and ranks them on a playability + style-closeness composite. An initial queue sweeps seven single-mapper cohorts and three bucket blends in roughly ten hours.

### Stack

- Python 3.12, PyTorch (nightly cu128 for RTX 5090 / sm_120)
- Lightning for training, Hydra for configs
- librosa for BPM / structure features, soundfile for audio I/O
- ruff, pytest (241-test suite)
- Managed with `uv` (`pyproject.toml` + `uv.lock`)
